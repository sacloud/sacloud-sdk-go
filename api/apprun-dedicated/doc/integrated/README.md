- AppRun 専有型においては、専用のサービスプリンシパルが必要
- 結合テストの中でサービスプリンシパルを都度作成することは技術的には可能
- しかし「サービスプリンシパルを作成できる」という権限は「AppRun 専有型を実行できる」という権限より強い
- そこで結合テストからサービスプリンシパルを作成するのは過剰な権限を要求するよくないデザインであろうと思慮
- 代わりに、結合テストの前提条件として、サービスプリンシパルをあらかじめ作成しておくことにする
- 作成はterraformにより実施

  ```console
  $ echo "TF_WORKSPACE=example" > .env
  $ mise plan
  $ mise apply
  ```

  `TF_WORKSPACE` と同名のusacloud profileが事前に必要

- なおコンテナレジストリもあれば使うことができる
- しかし必須ではないため一旦スキップ、今後の課題とする
